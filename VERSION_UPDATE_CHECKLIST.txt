================================================================================
                    VERSION UPDATE CHECKLIST FOR MGTOOLS
================================================================================

CRITICAL: Follow this checklist EVERY TIME you update versions in GitHub!

================================================================================
STEP 1: UPDATE VERSION NUMBERS IN BOTH BRANCHES
================================================================================

For MAIN branch (stable):
--------------------------
1. MGTools.user.js line 4:
   // @version      X.X.X

2. MGTools.user.js line 615 (approximately):
   CURRENT: 'X.X.X',

3. README.md line 3:
   **Version X.X.X** | All-in-one assistant...

4. README.md line 5:
   ![MGTools](https://img.shields.io/badge/version-X.X.X-blue)

5. CHANGELOG.md - Add new entry at top:
   ## Version X.X.X (YYYY-MM-DD)

For LIVE-BETA branch:
---------------------
Same 5 locations as above, but with BETA version number


================================================================================
STEP 2: VERIFY BRANCH DETECTION URLs (CRITICAL!)
================================================================================

MAIN branch must have:
---------------------
Line 7:  // @updateURL    https://github.com/Myke247/MGTools/raw/refs/heads/main/MGTools.user.js
Line 8:  // @downloadURL  https://github.com/Myke247/MGTools/raw/refs/heads/main/MGTools.user.js

LIVE-BETA branch must have:
---------------------------
Line 7:  // @updateURL    https://github.com/Myke247/MGTools/raw/refs/heads/Live-Beta/MGTools.user.js
Line 8:  // @downloadURL  https://github.com/Myke247/MGTools/raw/refs/heads/Live-Beta/MGTools.user.js

⚠️  WARNING: If these URLs are wrong, the branch detection will fail and
    users will see the wrong branch label (STABLE vs BETA)!


================================================================================
STEP 3: VERSION CHECKER SYSTEM (HOW IT WORKS)
================================================================================

The version checker automatically:
1. Detects which branch the user is on by checking @updateURL
2. Fetches BOTH stable and beta versions from GitHub
3. Compares user's current version against their branch
4. Displays color-coded indicator:
   - STABLE up-to-date: Bright green (#00ff00)
   - BETA up-to-date: Orange (#ff9500)
   - Update available: Red (stable) or Magenta (beta)

The tooltip always shows:
- Current version and branch (STABLE or BETA)
- Status (UP TO DATE, UPDATE AVAILABLE, or DEV VERSION)
- Both GitHub versions (Stable: vX.X.X | Beta: vX.X.X)
- Instructions for clicking/shift-clicking


================================================================================
STEP 4: TESTING CHECKLIST
================================================================================

After pushing updates, verify:

□ Main branch shows correct version number
□ Live-Beta branch shows correct version number
□ Main branch has @updateURL pointing to main
□ Live-Beta branch has @updateURL pointing to Live-Beta
□ Install from main - should show green and (STABLE)
□ Install from Live-Beta - should show orange and (BETA)
□ Hover tooltip shows both stable and beta versions
□ Version checker correctly identifies when updates are available


================================================================================
STEP 5: COMMON MISTAKES TO AVOID
================================================================================

❌ DON'T copy entire file from one branch to another and just change version
   (This will copy the wrong @updateURL!)

❌ DON'T forget to update all 5 version locations (script, config, readme, badge, changelog)

❌ DON'T push to GitHub without verifying @updateURL and @downloadURL

❌ DON'T skip testing - always verify color and branch label

✅ DO update versions in both branches separately

✅ DO verify URLs match the branch name

✅ DO test by installing from GitHub after pushing

✅ DO check that tooltip shows correct branch label


================================================================================

Branch detection URLs (2 locations per branch):

================================================================================
STEP 7: VERSION NUMBERING STRATEGY
================================================================================

Stable (main):     X.Y.Z
Beta (Live-Beta):  X.Y.Z+1  (usually one version ahead)

Example:
- main: 3.9.2
- Live-Beta: 3.9.3

After testing beta and merging to stable:
- main: 3.9.3
- Live-Beta: 3.9.4 (or next feature version)
